# GMind macOS App v2 — 设计文档

> CLI 给 Agent，App 给人。

> 状态说明：这是 macOS App v2 设计稿。当前 SwiftUI App 已实现菜单栏、记一条、问 AI、模型配置、最近列表、Taotie 面板和自动启动 `gmind serve`；未实现完整 Dashboard，也没有仓库内置的全局快捷键注册。快捷入口目前主要由菜单栏触发。

---

## 一、产品定位

| | CLI | Mac App |
|--|-----|---------|
| **用户** | AI Agent | 人 |

## 二、核心功能

### 2.1 记一条（Quick Record）

**入口：**
- 菜单栏 🧠 → 📝 记一条
- 全局快捷键 `⌘⇧A`

**面板：** 只有一个输入框。

```
┌──────────────────────────────────┐
│           🧠 记一条               │
├──────────────────────────────────┤
│  ┌────────────────────────────┐  │
│  │ 今天开了一下午会，讨论产品   │  │
│  │ 路线图。几个要点：           │  │
│  │ 1. Q3 重点做 AI 助手        │  │
│  │                             │  │
│  └────────────────────────────┘  │
│        [⌘↵ 记住了]               │
└──────────────────────────────────┘
```

**无选项：**
- 无标题输入框 → LLM 自动生成
- 无标签选择 → LLM 自动提取
- 无 Source 输入 → 后续支持 URL 粘贴
- 无 Auto-extract toggle → 默认开启

**快捷操作：**
- `⌘↵` = 保存
- `Esc` = 取消
- 打开时自动检测剪贴板，有文字自动贴入

**保存后反馈：**
- 菜单栏图标 🧠 → ✅（闪 1 秒）→ 🧠
- 后台默默 embedding + 实体提取

---

### 2.2 问 AI（Ask AI）

**唯一的信息出口。** 人不搜索文件，人只问问题。

```
┌────────────────────────────────────────────┐
│           🧠 问 AI                          │
├────────────────────────────────────────────┤
│  [我们 Q3 产品路线图讨论了哪些要点？]  ↵     │
├────────────────────────────────────────────┤
│  根据你的笔记，Q3 路线图的讨论涉及：         │
│                                            │
│  1. AI 助手集成                             │
│     在 [[ai-assistant-plan]] 中提到...      │
│                                            │
│  来源：ai-assistant-plan · onelap-global    │
└────────────────────────────────────────────┘
```

**设计要点：**
- 返回组织好的答案，不是文件列表
- `[[slug]]` 显示为可点击笔记标题
- 点击用系统默认编辑器打开

### 2.4 系统通知

**所有写入知识库的操作都通知人：**

| 来源 | 触发 | 通知内容 |
|------|------|---------|
| Chrome 插件 | 保存网页 | "🧠 网页已保存：《标题》" |
| CLI `gmind add` | 添加笔记 | "🧠 笔记已保存：标题" |
| CLI `gmind ingest` | 批量导入 | "🧠 导入完成：N 个文件" |
| 饕餮 Watcher | 自动导入 | "🧠 新文件入库：文件名" |
| 饕餮扫描 | 手动导入 | "🧠 导入完成：N 个文件" |

**实现：** server.py / CLI Python 端调用 `osascript` 发送 macOS 通知。

### 2.5 模型配置

**不是完整的 Settings 面板，只是一个轻量配置弹窗。**

```
┌──────────────────────────────────┐
│ 🔧 LLM 模型配置                   │
├──────────────────────────────────┤
│  Provider: [SiliconFlow ▼]       │
│  Model:    [DeepSeek-V4-Flash]   │
│  API Key:  [sk-****]             │
│  Base URL: https://api.silicon...│
│                                  │
│        [测试连接]  [保存]          │
└──────────────────────────────────┘
```

- 配置写入 `~/.gmind/config.toml` 的 `[llm]` 段
- 不配置则 `ask` 和 `enrich` 不可用

### 2.6 饕餮盛宴（Taotie）

**"吃光所有文件。"**

#### 2.6.1 全电脑扫描

扫描全电脑，找出可入库的文件，一键导入。

```
🍽️ 饕餮盛宴 — 全电脑扫描

扫描到 1,247 个候选文件

  📄 Markdown .......... 342 个
  📄 Word ..............  28 个
  📄 PDF ...............  14 个

  ┌──────────────────────────────┐
  │ ☑️ ~/笔记/产品规划.md         │
  │ ☑️ ~/笔记/需求文档.docx       │
  │ ☐  ~/Downloads/发票.pdf       │
  │ ...                          │
  └──────────────────────────────┘

  [全选]  [智能推荐]  [导入选中]
```

**一期支持格式：** `.md`, `.docx`, `.pdf`

**智能推荐：** LLM 判断文件是否"像知识"，排除日志/缓存。

#### 2.6.2 文件夹监听（Watcher）

用户设置文件夹，后台监听新增/修改，自动入库。

```
📁 ~/Documents/笔记          [移除]
📁 ~/Projects/知识库          [移除]
📁 ~/Downloads/文章           [移除]

[+ 添加文件夹]

扫描间隔：每 30 分钟
自动导入新文件：☑️
自动更新修改的文件：☑️
忽略：*.tmp, *.log, .git/
```

**技术：** macOS `FSEvents` API。

## 四、通知覆盖范围

```
所有写入操作 → macOS 通知
├─ Chrome 插件 /add        → "网页已保存"
├─ CLI gmind add           → "笔记已保存"
├─ CLI gmind ingest        → "导入完成：N 个"
├─ CLI gmind enrich        → "知识增强完成"
├─ 饕餮扫描导入            → "导入完成：N 个"
└─ 饕餮 Watcher 自动导入    → "新文件入库：文件名"
```
